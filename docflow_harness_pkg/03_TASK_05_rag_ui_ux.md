# Task 05 - Knowledge / RAG UI·UX

## 목표
사용자가 업로드와 참고 범위를 분리해서 이해하고, 어떤 문서를 근거로 생성했는지 직관적으로 확인할 수 있게 한다.

## 완료 조건
- 사이드바에 Knowledge 메뉴가 생긴다.
- Knowledge 화면에서 파일 카드, 태그, 업로드 날짜, 인덱싱 상태를 볼 수 있다.
- 생성 화면에서 참고 모드와 참고 자료 선택 UI가 제공된다.
- 결과 화면에서 참고 문서 목록이 보인다.
- 가능하면 근거 문단도 펼쳐서 볼 수 있다.
- style mode / strength 선택 UI가 존재한다.

## 핵심 UX 원칙
- 업로드와 참고는 분리
- 참고 범위는 사용자가 제어
- 어떤 문서를 참고했는지 항상 표시

## 구현 지시
1. Knowledge 라이브러리 화면 추가
2. 업로드 후 메타데이터 입력 및 인덱싱 상태 표시 연결
3. 문서 생성 화면에 참고 자료 선택 영역 추가
4. 참고 모드 추가
   - 자동 추천
   - 전체 라이브러리
   - 프로젝트 기반
   - 선택한 자료만
5. 참고 강도 / 출처 표시 옵션 추가
6. 결과 화면에 참고 자료 패널 추가
7. 가능하면 근거 문단 패널 추가
8. style mode / strength UI 추가

## Smart Default
- 문서 유형에 따라 기본 라이브러리 자동 선택
- workspace/project tag 기반 자동 선택 지원 가능하면 연결

## 주의사항
- UI만 만들고 실제 backend query/filter가 연결되지 않는 상태 금지
- 인덱싱 중 / 완료 / 실패 상태를 반드시 보여준다.
- 결과에 참고 문서가 없다면 왜 없는지 상태를 보여준다.

## 수동 테스트
- 파일 업로드 후 인덱싱 상태 변화 확인
- 참고 모드별 선택 반영 확인
- 결과 화면에서 참고 문서 표시 확인
- 출처 표시 on/off 확인
- style 설정값 전달 확인
